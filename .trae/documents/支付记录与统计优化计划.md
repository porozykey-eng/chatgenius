# 支付记录与统计优化计划

## 一、现状分析

### 数据库
- `orders` 表缺少 `alipay_trade_no`、`wechat_trade_no`、`refund_reason`、`refunded_at` 字段（admin.js 已引用但未建表）

### 后端 (admin.js)
- Dashboard 已有基础统计（今日/本周/本月/总收入、激活数、许可证数）
- 订单列表 API 已有筛选和分页，但返回字段缺少 `wechat_trade_no`
- 订单统计 API (`/orders/statistics`) 已有按类型和渠道分类

### 前端 (admin.html)
- 订单表格显示：订单号、套餐、金额、渠道、用户邮箱、状态、创建时间、操作
- 缺少：支付流水号显示、渠道统计图表

### 支付模块
- `alipay.js` 和 `wechat.js` 回调中会更新 `alipay_trade_no` / `wechat_trade_no`，但数据库字段不存在

---

## 二、需要修改的文件

### 1. 数据库迁移脚本
**文件**: `server/scripts/add-payment-fields.js`（新建）

添加缺失字段：
- `alipay_trade_no` VARCHAR(64) — 支付宝交易号
- `wechat_trade_no` VARCHAR(64) — 微信交易号
- `refund_reason` VARCHAR(255) — 退款原因
- `refunded_at` TIMESTAMP — 退款时间

### 2. 后端 admin.js
**文件**: `server/admin.js`

修改点：
- `/orders` 查询增加 `wechat_trade_no`、`refund_reason`、`refunded_at` 字段
- `/orders/:orderNo` 详情查询增加同样字段
- `/dashboard` 增加渠道收入统计（支付宝/微信各多少）
- `/dashboard/trends` 增加 `channel` 指标（按渠道统计订单数）

### 3. 前端 admin.html
**文件**: `server/admin.html`

修改点：
- 订单表格增加「支付流水号」列（根据渠道显示支付宝或微信交易号）
- Dashboard 增加「渠道收入分布」饼图（Chart.js）
- 订单表格增加「完成时间」列
- 订单详情弹窗（点击订单号查看完整信息，含流水号）

---

## 三、实施步骤

### Step 1: 创建数据库迁移脚本
- 新建 `server/scripts/add-payment-fields.js`
- 使用 `ALTER TABLE ... ADD COLUMN IF NOT EXISTS` 添加 4 个字段

### Step 2: 更新 admin.js 订单查询
- `/orders` 列表查询 SELECT 增加 `wechat_trade_no, refund_reason, refunded_at`
- `/orders/:orderNo` 详情查询同样增加
- 返回数据映射增加对应字段

### Step 3: 更新 admin.js Dashboard
- `/dashboard` 增加渠道统计：
  ```js
  const [channelStats] = await pool.query(
    "SELECT channel, COUNT(*) as cnt, SUM(CAST(price AS DECIMAL)) as total FROM orders WHERE status = 'completed' GROUP BY channel"
  );
  ```
- 返回 `channelStats` 给前端

### Step 4: 更新 admin.html 订单表格
- 表头增加「支付流水号」列
- 每行根据 `channel` 显示对应的交易号
- 增加「完成时间」列

### Step 5: 更新 admin.html Dashboard
- 在收入趋势图下方增加「渠道收入分布」饼图
- 使用 Chart.js doughnut 图表展示支付宝/微信收入占比

### Step 6: 增加订单详情弹窗
- 点击订单号弹出详情 Modal
- 显示：订单号、套餐、金额、类型、渠道、状态、创建时间、完成时间、支付流水号、用户邮箱、退款信息

---

## 四、验证方式

1. 运行迁移脚本，确认字段添加成功
2. 访问 admin 面板，确认订单列表显示支付流水号
3. 确认 Dashboard 显示渠道收入分布图
4. 点击订单号确认详情弹窗正常
5. 确认支付宝/微信回调能正确写入交易号
